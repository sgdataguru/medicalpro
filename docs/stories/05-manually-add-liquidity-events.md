# User Story: 5 - Manually Add Liquidity Events

**As a** Relationship Manager,
**I want** to manually add liquidity events that I am aware of into the system,
**so that** the platform reflects my on-the-ground knowledge and improves the overall intelligence for the team.

## Acceptance Criteria

*   An RM can create a new liquidity event entry for any prospect or company.
*   The form captures event type, affected individual/company, estimated timeline, and any supporting notes.
*   Manually added events are integrated into the same signal feed as system-detected events.
*   Manually added events are distinguished from system-detected events (e.g., labeled "Added by RM").
*   The manually added event influences the prospect's lead score.

## Notes

*   The system should be two-way — both ingesting external data and accepting RM input.
*   This was explicitly discussed: "Can RM add liquidity event? Yes, we can have that feature."
