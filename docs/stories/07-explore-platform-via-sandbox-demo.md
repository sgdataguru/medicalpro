# User Story: 7 - Explore Platform via Sandbox with Synthetic Data

**As a** prospective hospital client,
**I want** to interact with a sandbox environment populated with realistic synthetic data,
**so that** I can evaluate the platform's predictive and simulation capabilities before committing to a purchase.

## Acceptance Criteria

*   The sandbox is pre-loaded with synthetic hospital data covering all five modules (staffing, bed allocation, supply chain, finance, anomaly detection).
*   The synthetic data is realistic enough to demonstrate meaningful predictions and simulation outputs.
*   The prospective client can run what-if scenarios, view dashboards, and explore analytics without affecting production data.
*   The sandbox experience mirrors the full product's user interface and functionality.
*   The sandbox can be reset to its default state after each demo session.

## Notes

*   Building the sandbox is the current primary development objective — it must be completed before client engagement begins.
*   Hershey emphasized the sandbox as a pre-sales tool: "Before I even get a client, we want to build a sandbox... so that when I go to the hospital I can show them how it looks like when you have Medical Pro."
*   The sandbox approach helps "overcome skepticism by letting clients play with what-if scenarios themselves."
*   Synthetic data must be crafted with the same field structure as real hospital data to ensure a seamless transition post-sale.
