# User Story: 6 - Monitor Prospect-Specific News Alerts

**As a** Relationship Manager,
**I want** to receive automated news alerts whenever my prospects or their companies appear in the media,
**so that** I don't have to manually monitor news channels and can stay informed about relevant developments in real time.

## Acceptance Criteria

*   The system monitors news sources for mentions of prospects by name and company association.
*   Alerts are filtered to the RM's assigned or tracked prospects (TAM-linked, not generic news).
*   Each alert shows the prospect name, news headline, source, and date.
*   Alerts are surfaced on the dashboard and optionally via notification.
*   The news feed is client-specific and relevant, avoiding generic financial news.

## Notes

*   RMs currently watch news channels manually and set up alerts individually — this feature automates that process.
*   The discussion mentioned: "Is it possible to have APIs of all the news alerts which are happening based on the prospects?"
*   For the demo, mock data can be used; actual news API integration (e.g., Bloomberg) can follow once charges are agreed upon.
